Adding an existing project to Github
  1. Open terminal, cd to project folder and type 
    git init
  2. Go ahead and add & commit the files.
  3. Open github in browser, create a new repo.
  4. Copy the Quick Setup HTTPS link (ex: https://github.com/brandicameron/demo-repo2.git)
  5. Back in terminal, type 
            git remote add origin https://github.com/brandicameron/demo-repo2.git
        git branch -M main
        git push -u origin main
    (In the future you'll only need to enter git push.)

Creating Branches 
  1. git branch
    to determine what branch you're on. 
  2. git checkout -b name-of-branch
    creates your new branch. 
  3. git checkout branch-name
    to switch between branches.
  4. Add and commit changes as usual.
  5. On your first push, do 
    git push --set-upstream origin name-of-branch
  6. When you've made all your changes to the branch, go to github in the browser & click the "Compare & pull request" button. Add a note (if needed) and click "Create pull request".
  7. To merge the branch with main, click "Merge pull request" and "Confirm".
  8. Back in the local project, do 
    git checkout main
    to switch back to the main branch. Don't freak, the new changes won't be visible yet...
  9. Do 
    git pull
    to pull the changes to your local machine.
  10. Delete the old branch with 
    git branch -d name-of-branch
Tips
  • If you're only adding/committing one file that's already been added before (it will say modified when doing git status), you can skip the 
    add .
    and just do 
    git commit -am
    to add and commit in one step. 
  • git diff branch-name
    shows the diff between branches. 
  • git merge
    to merge two branches locally. 
  • git reset name-of-file
    to undo a previously staged file. (or just 
    git reset
    to unstage all files you just staged) 
  • git reset HEAD~1
    goes back one commit (which undoes the commit/staging just made) 
  • git reset --hard commitID
    completely removes all staging and commits that came after. Use 
    git push -f origin main(or whatever branch)
    to get the remote repository up to date with local. 
  • git checkout -- filename
    to remove a file that hasn't been staged yet that you want to revert back to before you made any changes.
